# CourtVision Mobile

React Native (Expo) app for on-device shot tracking and real-time shooting analytics.

## Run

```bash
cd mobile
npm install
cp .env.example .env
npx expo start
```

Set backend URL with:

```bash
EXPO_PUBLIC_API_URL=http://<your-ip>:5000
```

## TensorFlow Lite Integration Notes

`src/services/tfliteModel.ts` currently contains a mock model with realistic latency and confidence behavior.

The repository now also includes a baseline TensorFlow Lite artifact generated by:

```bash
cd /Users/bretthaas/CourtVision
python3 ml/train_baseline_model.py
```

Artifacts land in:

- `ml/artifacts/baseline_shot_model.tflite`
- `ml/artifacts/baseline_shot_model_manifest.json`

There is also a real-data tabular trainer generated by:

```bash
cd /Users/bretthaas/CourtVision
python3 ml/train_shotdetail_model.py
```

Artifacts land in:

- `ml/artifacts/shotdetail_make_model_weights.npz`
- `ml/artifacts/shotdetail_make_model_preprocessing.json`
- `ml/artifacts/shotdetail_make_model_manifest.json`

To switch to real TFLite:
- Create a native module bridge in iOS/Android that loads the `.tflite` model.
- Keep the `TFLiteShotModel` interface so UI/state code remains unchanged.
- Return `result`, `confidence`, court coordinates, and `inferenceLatencyMs` per frame.

Current limitation:
- The exported baseline is trained on synthetic structured features and is useful for pipeline validation, not production-quality CV claims.
- The public-data shotdetail model is real-data-backed, but it still predicts from tabular shot metadata rather than live camera frames.

## Dashboard metrics shown in app

- Attempts, make/miss, FG%
- Average inference latency
- Current/best make streak
- Zone-level shooting heat map
- Data-driven session summary with real shot plotting and share output
- Summary sharing includes the secure viewer token needed to inspect the protected session
